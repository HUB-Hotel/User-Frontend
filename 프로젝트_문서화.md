# 호텔 예약 시스템 - 프론트엔드 개발 문서

## 📋 프로젝트 개요

호텔 및 숙소 예약 플랫폼의 프론트엔드 애플리케이션입니다. React 기반으로 개발되었으며, SCSS를 사용한 스타일링과 React Router를 통한 라우팅을 구현했습니다.

## 🛠 기술 스택

- **프레임워크**: React 18
- **라우팅**: React Router v6
- **스타일링**: SCSS (Sass)
- **아이콘**: react-icons (Feather Icons)
- **날짜 처리**: date-fns
- **날짜 선택**: react-day-picker

## 🗺 라우팅 구조

```jsx
/                    → Landing (메인 페이지)
/search              → SearchResults (검색 결과 페이지)
/favorites           → Favorites (찜한 숙소 페이지)
```

### URL 쿼리 파라미터 (SearchResults)

- `destination`: 목적지 (예: "서울, 대한민국")
- `checkIn`: 체크인 날짜 (YYYY-MM-DD 형식)
- `checkOut`: 체크아웃 날짜 (YYYY-MM-DD 형식)
- `rooms`: 객실 수
- `guests`: 투숙객 수

예시: `/search?destination=서울, 대한민국&checkIn=2025-12-02&checkOut=2025-12-04&rooms=1&guests=2`

## 📊 데이터 구조

### 숙소 데이터 (allHotelsData)

모든 숙소 데이터는 `SearchResults.jsx`에서 `export const allHotelsData`로 정의되어 있으며, 다른 컴포넌트에서도 사용됩니다.

```javascript
{
  id: number,                    // 고유 ID
  name: string,                   // 숙소명
  price: number,                 // 가격 (원)
  address: string,               // 주소
  destination: string,           // 목적지 (예: "서울, 대한민국")
  type: 'hotel' | 'motel' | 'resort', // 숙소 타입
  starRating: number,            // 별점 (1-5)
  amenitiesCount: number,        // 편의시설 개수
  reviewScore: number,            // 리뷰 점수 (0-5)
  reviewText: string,             // 리뷰 텍스트 (예: "Very Good")
  reviewCount: number,           // 리뷰 개수
  image: string,                 // 이미지 URL
  imageCount: number,             // 이미지 개수
  freebies: {                     // 무료 서비스
    breakfast: boolean,
    parking: boolean,
    wifi: boolean,
    shuttle: boolean,
    cancellation: boolean
  },
  amenities: {                    // 편의시설
    frontDesk: boolean,           // 24시 프론트데스크
    aircon: boolean,              // 에어컨
    fitness: boolean,             // 피트니스
    pool: boolean                 // 수영장
  }
}
```

## 🔑 주요 기능

### 1. 메인 페이지 (Landing)

- **HeroSection**: 메인 배너와 검색 카드
- **Highlights**: 랜덤으로 선택된 4개의 인기 숙소 추천
- **TravelCards**: 나라 추천 카드 (멜버른, 파리, 런던, 콜롬비아)
- **NewsletterSection**: 뉴스레터 구독 섹션

### 2. 검색 기능

#### 검색 카드 (SearchCard)
- 목적지 입력 (자동완성 드롭다운)
- 체크인/체크아웃 날짜 선택 (react-day-picker)
- 객실 및 투숙객 수 선택
- 검색 버튼 클릭 시 `/search` 페이지로 이동 (쿼리 파라미터 포함)

#### 검색 결과 페이지 (SearchResults)
- **SearchHeader**: 검색 조건 수정 가능한 검색 카드
- **SearchFilters**: 사이드바 필터
  - 가격 범위 슬라이더
  - 별점 필터 (0+, 1+, 2+, 3+, 4+)
  - 무료 서비스 체크박스 (조식포함, 무료주차, WIFI, 공항셔틀버스, 무료취소)
  - 편의시설 체크박스 (24시 프론트데스크, 에어컨, 피트니스, 수영장)
  - 전체 해제 버튼
- **탭 필터**: All, Hotels, Motels, Resorts
- **정렬 옵션**: 추천순, 가격: 낮은 순, 가격: 높은 순, 평점순
- **호텔 카드 목록**: 필터링된 숙소 목록 표시
- **더보기 버튼**: 추가 결과 로드 (4개씩)

### 3. 찜하기 기능

- **localStorage**를 사용하여 클라이언트 측에서 찜한 숙소 저장
- `HotelCard` 컴포넌트에서 하트 아이콘 클릭으로 찜하기/해제
- 찜한 숙소는 빨간색 하트로 표시
- `/favorites` 페이지에서 찜한 숙소 목록 확인

### 4. 나라 추천 기능

- `TravelCards` 컴포넌트에서 "나라보러가기" 버튼 클릭
- 해당 나라로 검색 결과 페이지로 이동
- 페이지 상단으로 자동 스크롤

## 🔌 백엔드 연동 가이드

### API 엔드포인트 제안

#### 1. 숙소 검색 API

```
GET /api/hotels/search
Query Parameters:
  - destination: string (예: "서울, 대한민국")
  - checkIn: string (YYYY-MM-DD)
  - checkOut: string (YYYY-MM-DD)
  - rooms: number
  - guests: number
  - type?: 'hotel' | 'motel' | 'resort' | 'all'
  - minPrice?: number
  - maxPrice?: number
  - minRating?: number
  - freebies?: string[] (예: ['breakfast', 'wifi'])
  - amenities?: string[] (예: ['frontDesk', 'pool'])
  - sortBy?: 'recommended' | 'price_low' | 'price_high' | 'rating'
  - page?: number
  - limit?: number

Response:
{
  hotels: Hotel[],
  total: number,
  page: number,
  limit: number
}
```

#### 2. 숙소 상세 정보 API

```
GET /api/hotels/:id

Response:
{
  id: number,
  name: string,
  price: number,
  address: string,
  destination: string,
  type: string,
  starRating: number,
  amenitiesCount: number,
  reviewScore: number,
  reviewText: string,
  reviewCount: number,
  images: string[],
  freebies: {...},
  amenities: {...},
  description: string,
  // 기타 상세 정보
}
```

#### 3. 찜하기 API

```
POST /api/favorites
Body: { hotelId: number }

DELETE /api/favorites/:hotelId

GET /api/favorites
Response: { hotelIds: number[] }
```

#### 4. 인기 숙소 추천 API

```
GET /api/hotels/recommended?limit=4

Response: { hotels: Hotel[] }
```

### 현재 프론트엔드 데이터 구조

현재는 `allHotelsData` 배열에 73개의 숙소 데이터가 하드코딩되어 있습니다. 백엔드 API가 준비되면:

1. `SearchResults.jsx`의 `allHotelsData`를 API 호출로 대체
2. `useEffect`와 `useState`를 사용하여 데이터 로딩
3. 로딩 상태 및 에러 처리 추가

예시:

```javascript
const [hotels, setHotels] = useState([]);
const [loading, setLoading] = useState(false);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchHotels = async () => {
    setLoading(true);
    try {
      const params = new URLSearchParams();
      if (destination) params.set('destination', destination);
      // ... 기타 파라미터
      
      const response = await fetch(`/api/hotels/search?${params}`);
      const data = await response.json();
      setHotels(data.hotels);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };
  
  fetchHotels();
}, [destination, checkIn, checkOut, /* 기타 의존성 */]);
```

## 📝 주요 컴포넌트 설명

### SearchCard
- 메인 페이지의 검색 카드
- 목적지, 날짜, 객실/투숙객 선택
- 검색 버튼 클릭 시 `/search`로 이동

### SearchHeader
- 검색 결과 페이지의 검색 헤더
- URL 쿼리 파라미터를 읽어서 상태 초기화
- 검색 조건 변경 시 URL 업데이트

### SearchFilters
- 사이드바 필터 컴포넌트
- 필터 변경 시 `onFilterChange` 콜백 호출
- 부모 컴포넌트(SearchResults)에서 필터링 로직 처리

### HotelCard
- 개별 숙소 카드 컴포넌트
- 찜하기 기능 포함 (localStorage 사용)
- 숙소 정보 표시 및 "보러가기" 버튼

### TravelCards
- 나라 추천 카드 섹션
- "나라보러가기" 버튼 클릭 시 해당 나라 검색 결과로 이동

## 🔄 상태 관리

현재는 React의 `useState`와 `useContext`를 사용한 로컬 상태 관리만 사용하고 있습니다. 필요시 Redux나 Zustand 같은 상태 관리 라이브러리 도입을 고려할 수 있습니다.

### localStorage 사용
